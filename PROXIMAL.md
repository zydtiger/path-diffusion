# Proximal Sampling for Masked Reconstruction

This project uses a RePaint-style proximal conditioning mechanism during reverse diffusion to solve an inpainting inverse problem.

## Setup

- Ground-truth image: `x0`
- Binary mask: `M` (`1` = known pixel, `0` = unknown pixel)
- Measurement: `y = M \odot x0`
- Reverse state at timestep `t`: `x_t`

The model is unconditional (standard DDPM UNet). Conditioning is injected only in the sampling loop.

## One Reverse Step (t -> t-1)

At each timestep `t`, the sampler does three parts:

1. Predict noise and `x0` from current state:

`eps_theta = model(x_t, t)`

`x0_hat = (x_t - sqrt(1 - alpha_bar_t) * eps_theta) / sqrt(alpha_bar_t)`

2. Compute unconditional DDPM posterior sample for unknown region:

`mu_t = c1 * x0_hat + c2 * x_t`

where

- `c1 = sqrt(alpha_bar_{t-1}) * beta_t / (1 - alpha_bar_t)`
- `c2 = sqrt(alpha_t) * (1 - alpha_bar_{t-1}) / (1 - alpha_bar_t)`

If `t > 0`:

`x_unknown ~ N(mu_t, posterior_var_t I)`

`posterior_var_t = beta_t * (1 - alpha_bar_{t-1}) / (1 - alpha_bar_t)`

If `t = 0`:

`x_unknown = mu_t`

3. Proximal/conditioning step for known region and mask merge:

If `t > 0`:

`x_known = sqrt(alpha_bar_{t-1}) * y + sqrt(1 - alpha_bar_{t-1}) * z`, with `z ~ N(0, I)`

If `t = 0`:

`x_known = y`

Then combine:

`x_{t-1} = M \odot x_known + (1 - M) \odot x_unknown`

This enforces data consistency on known pixels while letting the diffusion prior generate missing content.

## Why It Helps

- Standard sampling ignores measurements and generates unconstrained samples.
- Proximal conditioning keeps observed pixels consistent at every step.
- Unknown regions are still generated by the learned prior, so completions remain realistic.

## Notes

- This is the core RePaint-style conditioning (known-region resampling at each reverse step).
- Higher-quality inpainting often benefits further from RePaint time-jump resampling schedules (forward/backward jumps), which are not yet included in this minimal version.
